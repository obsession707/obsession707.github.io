写给自己的说明书

rehoni / 2020-06-20

私人仓库

用于推送博客。采用hugo。

1. 构建流程
  1. 在typora中写好文章
  2. 添加YAML front matter
  3. 放入此仓库的content文件夹中
  4. 图片放入static\images中，并且在博文中将路径修改为/images/xxx.jpg
  5. 使用hugo命令生成
  6. 使用脚本备份和部署
2. hugo使用

建议阅读 hugo 中文文档

以及ivy作者说明书

hugo --theme=ivy
#  hugo server (--theme=hyde --buildDrafts)
hugo server
# 部署方法不是采用的下边的方式
hugo --theme=hyde --baseUrl="http://coderzh.github.io/"

cd public
git init
git remote add origin https://github.com/coderzh/coderzh.github.io.git
git add -A
git commit -m "first commit"
git push -u origin master
3. 图片路径问题

  1. config.toml中设置baseurl，设置成‘/’即可。

  2. 在.md的图片路径中，以static为根目录写全路径。

  3. 假设测试baseurl为http://localhost:1313/，图片位置：/static/img/hugo/123.png，那么在Markdown中写作![图片说明](/img/hugo/123.png)。这时，在浏览器就可以看到图片了 。

  4. typora软件设置，这样在每个大类文件夹中都有一个img文件夹，生产新的博文之后把对应的图片拷贝到项目的static/img中去就行。

4. typora使用注意事项

  1. 二级标题是居中，三级及之后的标题居左。

  2. 类似​:star:之类的内嵌typora的表情图案，此主题不支持。

  3. 可以参考A Plain Markdown Post这篇文章提取标题。

5. 两个脚本

push.sh 用于备份当前仓库的文件。

deploy.sh 用于将生成的网站推送到github.io仓库。

sh xxxx.sh 执行脚本文件。

  1. en方面的内容还没有
  2. 补全关于、简历
  3. 将categories和tags拆分成cn和en
